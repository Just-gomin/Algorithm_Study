# 💡 Heap

완전 이진 트리(Complete Binary Tree) 기반의 자료구조로, 부모 노드와 자식 노드 간에 일정한 대소 관계를 유지한다.

## 📌 Heap의 특성

- **완전 이진 트리** 구조를 유지한다. (마지막 레벨을 제외한 모든 레벨이 가득 차 있고, 마지막 레벨은 왼쪽부터 채워진다.)
- 부모 노드와 자식 노드 사이에 항상 특정 대소 관계가 성립한다.
- 형제 노드 간의 대소 관계는 보장하지 않는다.
- **우선순위 큐(Priority Queue)** 를 구현하는 대표적인 자료구조이다.

## 📌 Heap의 종류

### Max Heap (최대 힙)

- 부모 노드의 값이 자식 노드의 값보다 항상 크거나 같다.
- RootNode에 가장 큰 값이 위치한다.

```text
       90
      /  \
    70    80
   / \   /
  50  60 75
```

### Min Heap (최소 힙)

- 부모 노드의 값이 자식 노드의 값보다 항상 작거나 같다.
- RootNode에 가장 작은 값이 위치한다.

```text
       10
      /  \
    20    15
   / \   /
  50  30 40
```

## 📌 배열을 이용한 Heap 표현

완전 이진 트리의 특성 덕분에 배열(Array)로 효율적으로 표현할 수 있다.

- 인덱스 `i`인 노드의 **왼쪽 자식**: `2 * i + 1`
- 인덱스 `i`인 노드의 **오른쪽 자식**: `2 * i + 2`
- 인덱스 `i`인 노드의 **부모**: `⌊(i - 1) / 2⌋`

```text
배열: [90, 70, 80, 50, 60, 75]
인덱스:  0   1   2   3   4   5

          90(0)
         /     \
       70(1)   80(2)
       / \     /
    50(3) 60(4) 75(5)
```

## 📌 Operations

### Insert (삽입)

1. 새로운 노드를 배열의 마지막(완전 이진 트리의 마지막 위치)에 추가한다.
2. 추가된 노드를 부모 노드와 비교하며 Heap 조건을 만족할 때까지 위로 이동시킨다. (**Heapify Up / Bubble Up**)

```text
Max Heap에 85 삽입:

[삽입 전]          [삽입 후 - Heapify Up]
      90                  90
     /  \                /  \
   70    80    →       70    85
  / \   /             / \   / \
50  60 75           50  60 75  80
                  (80과 85 교환)
```

- 시간 복잡도: `O(log n)`

### Extract (최댓값 또는 최솟값 추출)

1. RootNode(최댓값 또는 최솟값)를 꺼낸다.
2. 마지막 노드를 RootNode 자리로 이동시킨다.
3. 이동한 노드를 자식 노드와 비교하며 Heap 조건을 만족할 때까지 아래로 이동시킨다. (**Heapify Down / Bubble Down**)

```text
Max Heap에서 90 추출:

[추출 전]         [마지막 노드를 루트로]  [Heapify Down]
      90                75                    80
     /  \              /  \                  /  \
   70    80    →     70    80     →        70    75
  / \   /           / \                  / \
50  60 75          50  60               50  60
```

- 시간 복잡도: `O(log n)`

### Peek (최댓값 또는 최솟값 조회)

- RootNode의 값을 제거 없이 반환한다.
- 시간 복잡도: `O(1)`

## 📌 Heapify (힙 구성)

임의의 배열을 Heap으로 만드는 과정이다. 마지막 내부 노드(LeafNode가 아닌 가장 마지막 노드)부터 RootNode 방향으로 각 노드에 Heapify Down을 적용한다.

- 시간 복잡도: `O(n)` (단순히 n번 Insert하는 것보다 효율적)

## 📌 시간 복잡도

| Operation | 시간 복잡도 |
| :----------- | :----------- |
| Insert | `O(log n)` |
| Extract | `O(log n)` |
| Peek | `O(1)` |
| Heapify | `O(n)` |
| Search | `O(n)` |

## 📌 활용

- **우선순위 큐(Priority Queue)**: 우선순위가 높은 요소를 빠르게 꺼내야 하는 경우에 사용한다.
- **힙 정렬(Heap Sort)**: Heapify로 배열을 힙으로 만든 후, Extract를 반복하여 정렬한다. 시간 복잡도 `O(n log n)`.
- **최단 경로 알고리즘**: 다익스트라(Dijkstra) 알고리즘에서 방문할 노드를 선택하는 데 사용한다.
- **Top-K 문제**: N개의 데이터 중 가장 크거나 작은 K개를 효율적으로 구하는 데 활용한다.
